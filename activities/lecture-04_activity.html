Lecture 4: Machine learning

AI 100, 2026W1

Lecture overview

Learning outcomes

By the end of this lecture, you will be able to:

  • Explain how machine learning differs from explicit programming and search, and give an example of a problem better suited to learning rules from data.
  • Identify the features and labels in a supervised-learning dataset.
  • Describe how a model is trained on labelled examples to classify new, unseen examples.
  • Recognize that a linear model separates two classes with a straight-line boundary.
  • Assess, from data plotted by feature, whether two classes are linearly separable and how the number of features affects this.
  • Construct a decision tree that correctly separates a small labelled dataset into its classes.
  • Explain why data is split into training and testing sets, and compute a model’s accuracy on the testing set.
  • Explain why a model with high training accuracy may still fail to generalize to new data.

Differences from previous lectures

TipActivity: Think, pair, share
  • What kinds of problems would be hard to solve with explicit programming or search?
  • We do not always want to use pre-defined, set rules or states and actions like we did in explicit rule-based programming and search.
  • In machine learning, we build a model that will learn the rules from the data.

Credit card fraud detection

  • Have you ever had your credit card declined on vacation? You were probably interacting with a credit card fraud detection model!
  • Being able to identify fraudulent transactions is useful: it protects businesses and credit card holders from financial loss after fraud attempts happen.
  • Billions of transactions are processed every day, so being able to identify these transactions programmatically is even better.
TipActivity: Think, pair, share

What information could be used to identify whether a single transaction is fraudulent or not?

Features and labels

  • Data used to identify whether a transaction is fraudulent could include:
User ID Amount ($) Distance from Home (km) Account Age (days) Fraud?
1 120 25 1190 Legit
2 100 90 606 Fraud
3 105 55 582 Fraud
4 145 0 524 Legit
  • Each row in the table above is an example. Each example has features and labels.
    • Features are things we already know about an example, like User ID, Amount, Distance from Home, and Account Age.
    • Labels are things we would like to predict, like Fraud.
    • We represent the features with X and represent the labels with Y.
  • We train a model on lots of examples with labels we already know so that the model can learn to predict labels for new examples. Predicting a label is also called classification.
  • Thus, our inputs to a machine learning problem are features and our outputs are labels.
TipActivity: Discussion

How could you use each feature in the table to predict fraud? Are all of the features useful?

  • If you have lots of features, manually coming up with a rule for every possible example the model might see is not reasonable! With machine learning, the model learns the rules for you.
  • Open questions: What kind of rules does the model learn? How do the rules relate to the features and labels?

Linear models

  • We want to know whether a transaction is fraudulent or not based only on features.
  • An easy way to do this is to use a linear model: we draw a 2D line (or 3D plane) between the two types of transactions, say that everything on one side is fraudulent, and say that everything on the other side is legitimate.   • The more information we have about the transactions, the easier it is to draw a line or plane that separates them.
TipActivity: Discussion

Let’s consider two separator planes. Which one do you prefer? Why?

  • Some planes represent the relationship between features and labels better than others. We want to find the best one possible so that we can make good predictions.
  • Lines and planes are represented by parameters. During the training process, the linear model tries to learn the parameters that give us the best one.

Student mental health

  • Let’s consider another scenario: we have a list of students, and we want know whether they are struggling or thriving at university.
TipActivity: Think, pair, share

What might be some features we could use to make this prediction? What would the label be?

  • The features we’ll use are hours spent doing coursework, hours spent doing social activities, and whether or not the student landed an internship (we’ll use 1 to represent yes and 0 to represent no).
  • The label will be the group the student should belong to: struggling or thriving.
TipActivity: Discussion

Is it possible to draw a line or plane to perfectly separate the two groups? What kind of shape might do a better job?

Decision trees

  • Instead of using a linear model, we could use a decision tree to separate the two groups.
  • A decision tree is like a 20 questions game: we ask a series of yes/no questions about the features to determine which group a student belongs to.
TipActivity: Think, pair, share

What kinds of questions are more useful to ask at the start of a 20 questions game? What about later in the game?

  • Different questions will give us different decision trees, and some will do a better job of separating labels than others.
  • Instead of finding the line or plane that best separates the labels, a decision tree finds the questions that best separate the labels at each step of the game. These questions are the parameters of the decision tree.

Training and testing data

  • Once the model is deployed in the real world, we might see that the model doesn’t really match up with relationships and trends in the real world.
  • To give us an idea of how a model will perform in the real world before deploying it, we set aside part of the data (the testing set) before training the model with the rest of the data (the training set). We then see how well the model performs on the testing set.
  • Accuracy is the percentage of predictions the model gets right. For example, if there are 10 examples in the testing set, and the model correctly predicts the target for 8 of them, the model has an accuracy of 8/10 = 80%.
TipActivity: Think, pair, share

Why do you think it is so important that we don’t train the model on the testing set?

  • Since the model has never seen the testing set before, it should perform about as well on the testing set as it will on data in the real world.
